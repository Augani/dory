## LinkedIn

Introducing Dory 0.3.2: a local Linux workspace that assembles around you.

This release replaces fixed app editions with one smaller Docker Core and signed, removable components for Kubernetes, Linux Machines, the graphical desktop runtime, Debian, Ubuntu, and Kali.

That means developers can keep the foundation focused, install only the capabilities they need, run and test graphical Linux applications beside their containers, and remove optional payloads later without deleting workload data.

For agent-heavy workflows, Dory's preview sandbox command creates a dedicated Linux VM per run. It shares no host files by default, supports explicit read-only or read-write mounts, can enforce no-network execution, offers rollback and TTL cleanup, and deletes the sandbox by default. Agents get room to experiment without being handed the rest of the Mac.

We also removed about 112 MB of duplicate compatibility payload and added custom local-domain routing through Dory's trusted HTTP and HTTPS proxies.

Dory is open-source and built for Apple Silicon Macs.

Release: https://github.com/Augani/dory/releases/tag/v0.3.2
